# 观众提问墙 MVP

## 目的

创建一个用于提升展示型活动互动性的实时网页体验。观众加入活动后，可以提交问题、为自己也关心的问题点赞，并参与单选投票。主持人通过控制台管理活动，公开展示视图展示问题墙、投票结果和加入信息。

## 范围

产品建模的是活动，而不是底层的演讲或会议。活动由一个主持账号拥有，观众通过系统生成的访问码或链接加入，并按草稿、进行中、已结束的单向生命周期流转。已结束活动保持只读可访问，但不能重新开启。

## MVP 目标

- 主持账号可以创建并管理多个活动。
- 观众可以快速加入活动，默认匿名，也可以提供显示名。
- 观众视图、展示视图和主持控制台中的活动更新，应在无需手动刷新的情况下，于 1-2 秒内显示。
- 第一版以单个活动约 200 个活跃观众会话为规模目标。
- 交互模型要足够简单，适合现场演讲和分享场景。

## 角色与视图

### 主持账号

主持账号可以创建活动并使用主持控制台。第一版可以使用少量示例账号，而不提供完整的账号管理系统。

### 观众会话

观众会话是观众在单个活动中的临时身份。它保留在同一浏览器中直到活动结束，并保存显示名、问题点赞状态和投票选择。它不跨设备同步。

### 观众视图

观众使用该视图：

- 通过访问码或链接加入活动。
- 选择匿名进入，或提供显示名。
- 在活动的问题字数限制内提交观众问题。
- 为可见且未回答的问题点赞。
- 参与进行中的单选投票。
- 查看自己的投票选择，但不查看聚合投票结果。

### 主持控制台

主持人使用该视图：

- 创建活动。
- 将活动从草稿切换为进行中，再切换为已结束。
- 查看由自己的主持账号拥有的活动。
- 置顶或取消置顶问题。
- 标记问题已回答或未回答。
- 将隐藏问题作为低频安全控制手段。
- 创建、关闭和排序投票。
- 仅在投票收到选择前编辑或删除投票。
- 查看投票百分比和原始票数。
- 将活动数据导出为 CSV。
- 软删除活动。

### 展示视图

展示视图是公开的只读投屏界面。它组合展示：

- 加入信息，包括访问码和二维码。
- 按问题排序展示的可见且未回答的问题墙。
- 存在置顶问题时，对置顶问题进行视觉强调。
- 按投票排序展示的进行中投票和已关闭投票结果。

## 活动模型

活动包含：

- 标题。
- 状态：草稿、进行中、已结束。
- 系统生成的访问码。
- 创建时间。
- 拥有者主持账号。
- 问题字数限制，默认 240 字符。

已删除活动采用软删除：它从主持控制台默认列表中隐藏，且不再能通过观众视图或展示视图访问，但数据可以继续保留。

## 活动生命周期

- 草稿活动正在准备中，不接受观众互动。
- 进行中活动接受问题、问题点赞和投票选择。
- 已结束活动不再接受互动，但仍可通过访问码只读访问。
- 已结束活动不能重新开启。后续或重复场次应创建新的活动。

## 问题

观众问题是纯文本，并带有观众会话中的可选显示名。观众提交问题后不能编辑或删除。

问题排序：

- 主列表展示可见且未回答的问题。
- 置顶问题显示在正常排序之前。
- 其余问题按点赞数降序排列，点赞数相同时按提交时间升序排列。
- 已回答问题离开观众视图和展示视图的主列表。
- 隐藏问题不出现在观众视图和展示视图中。

问题点赞：

- 表示某个观众会话也关心这个问题。
- 不支持反对票。
- 可以取消，但取消不是第一版优先能力。
- 同一个观众会话对同一个问题最多贡献一个点赞。

## 投票

第一版投票仅支持单选。每个观众会话可以为一个投票选择一个选项，并且可以在投票关闭前修改选择。

一个活动中可以有多个进行中投票。产品不对进行中投票数量设置上限。

投票排序：

- 主持控制台可以手动调整进行中投票的顺序。
- 默认新创建的投票排在前面。
- 观众视图和展示视图使用同一套投票排序。

投票可信度：

- 投票收到选择前，可以编辑投票题干和选项。
- 投票收到选择前，可以删除投票。
- 投票一旦收到选择，就只能关闭，不能再改题干、改选项或删除。

投票结果可见性：

- 观众视图只显示该观众会话自己的选择。
- 展示视图显示百分比结果。
- 主持控制台显示百分比和原始票数。

## 导出与保留

活动数据会一直保留，直到拥有者主持账号手动删除。MVP 支持用于活动后复盘的 CSV 导出，包含观众问题、问题点赞、投票、投票选项和投票结果。

## 非目标

- 建模演讲、场地、议程、演讲者简介或完整会议管理。
- 完整账号管理流程，例如找回密码、团队、组织、邀请和共享活动所有权。
- 问题显示前的主持人审核。
- 观众侧编辑或删除已提交问题。
- 自动识别或合并重复问题。
- 单选之外的投票类型。
- 在观众视图中显示聚合投票结果。
- 完整国际化；第一版 UI 可以是中文。
- 在产品模型中承诺长连接、轮询或任何具体刷新实现。
- PDF 报告、分析仪表盘或邮件发送。

## MVP 成功标准

- 主持账号可以创建活动，并让活动经过草稿、进行中、已结束状态。
- 观众可以通过二维码或访问码加入，默认匿名，也可以提供显示名。
- 观众可以在活动字数限制内提交问题、为问题点赞，并参与单选投票。
- 展示视图无需手动刷新即可展示加入信息、排序后的问题墙、置顶问题强调和实时投票百分比结果。
- 主持控制台可以管理问题、管理投票、查看结果、导出 CSV、结束活动和软删除活动。
- 已结束活动只读且不能重新开启。
- 已删除活动不再公开可访问。
